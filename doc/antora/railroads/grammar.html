query_system_time_period_specification

All tables in XTDB are system-versioned. The System Time period specification has four forms: 
  1. AS OF ...
    - Acts as a point-in-time time-traveling mechanism. A query run with this specification will retrieve the rows that were current at the chosen point in time.
  2. BETWEEN ... AND
    - Returns all rows between the two times provided, inclusive of both. 'Between' acts as a closed-closed period model.
  3. FROM ... TO
    - Returns all rows between the two times provided, inclusive of the FROM time but not the TO time. 'From/To' acts as a closed-open period model.
  4. FOR ALL
    - Returns rows for all available system time. 'For All' will return every version of each row.
Without any of these forms, XTDB queries will, by default (assuming `AS_OF_NOW`), return rows "as of now".
